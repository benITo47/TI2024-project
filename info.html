About Pathfinding Visualizer

The Pathfinding Visualizer is a tool designed to help users understand and visualize how various graph search algorithms work. This tool supports two modes:

  • 2D Grid Mode: A grid-based visualization where algorithms find paths from a start point to an endpoint.
  • Graph Mode: A general graph visualization with nodes and edges, ideal for non-grid-based structures.

How It Works

The visualizer allows you to select start and end points, place walls or weights, and then run your chosen algorithm to see how it searches for a path. Each step of the algorithm is animated to provide an intuitive understanding of its mechanics.

In grid mode, the grid cells represent nodes, and adjacent cells are connected. In graph mode, nodes and edges are represented as circles and lines, respectively, with longer edges representing bigger weights.

Implemented Algorithms

Breadth-First Search (BFS)

BFS explores all neighbors at the current depth before moving deeper. It guarantees the shortest path. BFS is ideal for scenarios where finding the shortest path is critical.

Depth-First Search (DFS)

DFS dives deep into one branch before backtracking. It does not guarantee the shortest path but is useful for exploring all possible paths in a graph.

Dijkstra's Algorithm

Dijkstra's algorithm finds the shortest path in weighted graphs. It uses a priority queue to explore nodes in order of their accumulated cost from the start node.

A* Algorithm

A* is a heuristic-based search algorithm that combines the strengths of Dijkstra's algorithm and a heuristic function. It is faster than Dijkstra when a good heuristic is provided, and it guarantees the shortest path.
